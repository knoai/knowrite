# knowrite 小说创作系统 — 核心优势详解

## 一、多 Agent 协作流水线：不是写，是生产

传统 AI 写作是"一个模型一次性出稿"，质量完全取决于单次 Prompt 的效果。knowrite 将写作拆解为一条**角色分工明确的生产线**：

```
作者初稿 → 编辑评审（不通过则改稿循环）→ 去AI化 → 校编 → 读者反馈 → 摘要存档
   ↑___________________________________________|
                （最多 3 轮循环改稿）
```

每个角色有专门的 Prompt 模板和温度配置：
- **作者**：高温（0.85），创意优先
- **编辑**：中温（0.8），结构+逻辑审查
- **去 AI 化**：高温（0.85），消除 AI 腔调
- **校编**：低温（0.2），错别字+标点+语病
- **读者**：0 温，客观评分+反馈
- **摘要器**：0 温，压缩提炼关键信息

**价值**：7 个 Agent 自动完成审稿、改稿、润色、校对，全流程无人值守。你设置好大纲和风格，系统自动产出可用章节。

---

## 二、双模式创作：量产与探索兼顾

| | 工业风 | 自由风 |
|---|---|---|
| **规则数量** | 8~33 条硬性规则 | 2 条底线（吸引力+人物一致性） |
| **大纲约束** | 严格遵循，偏离自动检测矫正 | 大纲为参考，允许偏离和即兴 |
| **编辑轮次** | 最多 3 轮严格评审 | 1 轮轻量检查 |
| **去 AI 化** | 重度 + 全量校编 | 轻度，跳过 proofreader |
| **输出速度** | 较慢（质量优先） | 较快（创意优先） |
| **适用场景** | 商业化量产、IP 开发、长篇连载 | 灵感驱动、探索性写作、短篇实验 |

**价值**：同一套系统，既能跑严格的出版级流程，也能跑轻快的创意探索流程。按项目需求一键切换。

---

## 三、智能上下文管理：百章长篇不崩

AI 写长篇小说的最大难题是**上下文遗忘**。knowrite 采用三层上下文策略：

| 层级 | 范围 | 处理方式 | 目的 |
|------|------|---------|------|
| **近史** | 前 1~4 章 | 保留全文 | 确保衔接自然 |
| **中史** | 近 5 章 | 保留摘要 | 掌握近期剧情走向 |
| **远史** | 更早章节 | 合并压缩为提要 | 防止设定遗忘，控制 Token 消耗 |

额外机制：
- **反重复提醒**：自动检测近史中已出现的情节元素，提醒作者避免重复
- **世界观注入**：每个创作步骤自动注入世界观记忆库（lore、人物、地图、剧情线）

**价值**：实测支持 50+ 章长篇小说，设定前后一致、情节不重复、人物不 OOC。

## 四、Prompt 工程化：不是调 Prompt，是进化 Prompt

系统内置**Prompt 模板引擎**：

- **变量替换**：`{{style}}`、`{{topic}}`、`{{outlineDetailed}}` 等自动注入
- **Include 机制**：`{{include:core-rules}}` 复用公共规则块
- **模式适配**：工业风加载 `writer.md`，自由风自动回退到 `writer-free.md`
- **动态注入**：世界观上下文、反重复提醒、平台风格、作者风格实时拼接

更关键的是 **Prompt 进化系统**：

1. 每章产出 Fitness 评分（钩子、节奏、人设、画面感等 33 维度）
2. 低 Fitness 样本自动进入进化数据集
3. 系统生成多个 Prompt 变体，在低 Fitness 样本上测试
4. 选择 Fitness 提升最大的变体，一键应用到生产环境

**价值**：Prompt 质量不再靠人工反复调试，而是靠数据自动迭代优化。

## 五、33 维度质量评估体系

不是"感觉这章写得不错"，而是**可量化的质量数据**：

### 评审维度（支持 8/15/33 维三档预设）

**开篇与结构（5 维）**：黄金三章、开篇钩子、章末钩子、期待感、结构完整性
**人物塑造（6 维）**：主角动机、主角缺陷、成长弧光、反派塑造、配角记忆点、人设一致性
**情节与叙事（7 维）**：主线清晰度、爽点设计、反转意外性、悬念铺设、伏笔回收、冲突密度、情节推进效率
**情绪与节奏（5 维）**：情绪传递、情绪曲线、节奏张弛、信息密度、水文控制
**文笔与语言（5 维）**：画面感、对话质量、语言风格适配、修辞水准、错别字与语病
**世界观与设定（3 维）**：设定新颖度、设定一致性、设定展示方式
**商业价值（2 维）**：平台调性匹配、品类辨识度

### Fitness 评分

每章自动生成 0~1 的综合 Fitness 分数，叠加雷达图展示各维度强弱。支持：
- **历史趋势追踪**：Fitness 随章节变化曲线
- **低分自动告警**：低于阈值触发重试或人工干预
- **Prompt 进化驱动**：低 Fitness 样本自动进入进化数据集

**价值**：创作质量从"玄学"变成"工程指标"。

## 六、作家轮换：打破模型风格固化

同一模型连续写 10 章，文风会越来越雷同。knowrite 支持**按章节轮询不同模型**：

```
第 1 章 → 模型 A（deepseek-v3）
第 2 章 → 模型 B（deepseek-r1）
第 3 章 → 模型 C（qwen-max）
第 4 章 → 模型 A（循环）
```

配置界面支持：
- 任意数量模型参与轮换
- 每个模型独立配置 provider、model、temperature、alias
- 同一章内的初稿与改稿固定使用同一模型，保证风格一致

**价值**：长篇小说不再是"同一个 AI 在自言自语"，而是多个"作家"交替执笔，文风更有层次。

---

## 七、世界观记忆库：写到第 50 章设定不崩

长篇小说最容易崩的是**世界观一致性**。系统提供完整的世界观管理工具：

| 模块 | 功能 |
|------|------|
| **Lore 库** | 力量体系、历史事件、种族设定、神器道具等 lore 条目 |
| **人物面板** | 每个角色的外貌、性格、目标、背景、关系图谱 |
| **地图系统** | 区域划分、地理连接、势力分布 |
| **剧情线** | 主线/支线/暗线管理，每章标注剧情节点 |
| **套路模板** | 可复用的故事结构模板（三幕式、英雄之旅、退婚流等） |

**自动注入**：每个创作步骤（大纲、初稿、编辑）的 Prompt 中自动拼接世界观上下文，确保 AI 始终记得"这个世界是怎么运转的"。

**价值**：百万字长篇的设定管理从人脑记忆变成系统化管理。

---

## 八、多模型统一接入：不被任何厂商绑定

系统通过统一的 `BaseProvider` 接口接入任意模型：

**内置直连（API Provider）**：
- OpenAI 兼容接口（百炼、硅基流动、OneAPI 等）
- Ollama（本地开源模型）
- LM Studio（本地模型 GUI）

**本地代理转发（Web Provider）**：
- 通过 `PROXY_URL` 转发到用户自建的本地代理
- 代理服务负责实际的浏览器自动化或网页 API 调用
- knowrite 本身不耦合任何特定 web provider 的实现

**价值**：
- 模型切换零成本：改一行配置即可换模型
- 不被单一厂商绑定：成本上涨随时迁移
- 本地隐私保护：敏感题材可用本地 Ollama 完全离线运行

## 九、可扩展架构：每个环节都可自定义

| 扩展点 | 方式 |
|--------|------|
| **Prompt 模板** | 在 `prompts/` 目录添加 `.md` 文件，系统自动识别 |
| **评审维度** | 在设置页自定义维度名称和描述，即时生效 |
| **作者风格** | 80+ 内置风格，支持自定义添加 |
| **平台风格** | 10+ 内置平台（番茄/起点/晋江等），支持自定义 |
| **角色模型配置** | 每个 Agent 角色可独立指定 provider、model、temperature |
| **Provider 扩展** | 实现 `BaseProvider` 接口即可接入新模型 |

**价值**：不是一套固定流程，而是一个可定制的创作框架。你的创作方法论可以沉淀为系统配置。

## 十、完整的工程化工具链

| 工具 | 功能 |
|------|------|
| **大纲偏离检测** | 自动检测章节是否偏离大纲，一键矫正重写 |
| **风格迁移** | 已写章节一键切换平台风格或作者风格 |
| **重复检测修复** | 检测内容重复（情节/台词/设定），自动修复 |
| **导入外部作品** | 导入已有小说文本，自动分章并生成摘要索引 |
| **导入外部大纲** | 导入已有大纲，自动优化并启动创作 |
| **多卷管理** | 支持多卷长篇小说，每卷独立纲章 |
| **执行追踪** | 每个 Agent 调用的输入/输出/耗时自动记录 |

---

## 总结：这不是一个工具，是一套方法论

knowrite 的核心价值不在于"让 AI 写小说"，而在于**把小说创作从艺术创作转变为可工程化管理的生产流程**。

- **质量可控**：33 维度评审 + 循环改稿 = 下限有保障
- **规模可控**：智能上下文 + 世界观记忆 = 长篇不崩
- **成本可控**：Prompt 进化 + Fitness 评分 = 持续优化投入产出比
- **风格可控**：作家轮换 + 双模式切换 = 量产与创意兼顾

如果你用单模型写过小说，你会明白这些痛点的真实存在。knowrite 就是为解决这些痛点而生的。
