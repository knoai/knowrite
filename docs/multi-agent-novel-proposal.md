# Multi-Agent 小说创作优化方案

> 基于现有 `novel-engine` 架构，引入 6 个专业角色分工协作，提升长篇章节的文笔自然度、结构合理性与读者体验。
> 本方案新增支持 **热血磅礴风（热血玄幻）** 与 **10后向（热血中二轻小说）** 两种特色风格库。

---

## 1. 角色定义

| 角色 | 英文名 | 职责 | 输出文件 |
|------|--------|------|----------|
| **策划** | Planner | 世界观、主题大纲、详细分章大纲、人物设定 | `outline_theme.txt` / `outline_detailed.txt` |
| **作者** | Writer | 按大纲与上下文撰写章节初稿 | `chapter_{N}_raw.txt` |
| **编辑** | Editor | 审查情节连贯性、人物一致性、节奏问题，提出修改意见 | `chapter_{N}_edit.txt` |
| **作者-改** | Writer-Reviser | 根据编辑意见修改稿件 | `chapter_{N}_edited.txt` |
| **去AI化** | Humanizer | 去除 AI 腔、统一个人文风、增强画面感与口语化 | `chapter_{N}_humanized.txt` |
| **校编** | Proofreader | 错别字、标点、语法、格式规范 | `chapter_{N}_final.txt` |
| **读者** | Reader | 从读者视角评分与反馈（爽点/毒点/期待感） | `chapter_{N}_feedback.json` |
| **摘要** | Summarizer | 提取本章核心剧情点，供后续章节滚动窗口使用 | `chapter_{N}_summary.txt` |

---

## 2. 工作流（按章流水线）

### 2.1 全局阶段 `startNovel`
仅由 **策划** 完成：
1. 生成 `outline_theme.txt`（主题大纲）
2. 生成 `outline_detailed.txt`（详细分章大纲）

### 2.2 每章阶段 `continueNovel`
对第 N 章依次执行：

```
作者(raw)
   ↓
┌──────────────────────────┐
│ 编辑(edit)               │
│ 如果评审不通过 ──→ 作者改(edited)
│ 再次编辑评审（最多3轮）  │
└──────────────────────────┘
           ↓ 评审通过后
      去AI化(humanized)
           ↓
      校编(final)
           ↓
      读者(feedback)
           ↓
      摘要(summary)
```

**上下文规则**：
- **作者 / 作者改**：使用「滚动摘要窗口」—— 前 1 章 `final` 全文 + 更早章节的 `summary`（最近 3 章）+ 双重大纲。
- **编辑**：除当前稿件外，额外注入前 1 章 `final` 全文，确保跨章连贯性审查。
- **去AI化 / 校编 / 读者 / 摘要**：仅处理当前章节内容，不注入跨章上下文（专注单章质量）。

**编辑-作者改循环规则**：
1. 作者完成初稿后，进入编辑评审。
2. 编辑必须在文末明确输出评审结论：\n   `【评审结果：通过】` 或 `【评审结果：不通过】`
3. 如果评审不通过，且未达到最大轮次（默认 3 轮），则返回作者改稿，改稿后再交编辑评审。
4. 如果评审通过，或已达到最大轮次，则进入后续流程（去AI化 → 校编 → 读者 → 摘要）。
5. 中间轮次的编辑意见和修改稿会归档为 `chapter_{N}_edit_v1.txt`、`chapter_{N}_edited_v1.txt` 等，最终版本保留为标准文件名 `chapter_{N}_edit.txt` 和 `chapter_{N}_edited.txt`。

## 4. 风格库（新增）

### 4.1 热血磅礴风（热血玄幻）
**核心特征**：
- 世界观宏大，格局横跨诸天万界或万古长河
- 悬念重重，伏笔深远，常出现"上古秘辛"、"惊天大密"
- 人物名字带有古风霸气（如叶凡、石昊、楚风）
- 打斗场面气势磅礴，动辄"星河倒卷"、"举世皆惊"
- 主角常有霸气侧漏的台词与不屈意志
- 节奏上喜欢"镇压一切敌"、"同级无敌"、"逆天而行"

**Prompt 注入关键词**：
> "请模仿热血磅礴的玄幻风格：大气磅礴、悬念重重、伏笔深远。人物名号响亮霸气，打斗场面要有'举世皆惊'、'震动天下'的宏大感，主角台词不屈霸气。"

### 4.2 10后向（热血中二轻小说）
**核心特征**：
- 面向 10-15 岁读者，节奏极快，三章一小爽、五章一大爽
- 人设鲜明标签化（如"签到系统"、"无敌流"、"满级大佬"）
- 热血中二，常出现燃向台词、玩梗、网络流行语
- 二次元元素（如转生、异能学院、异世界冒险）
- 对话轻松幽默，避免过于沉重的伦理探讨
- 爽点密集：打脸、逆袭、藏拙、震惊全场

**Prompt 注入关键词**：
> "请面向10后读者创作：节奏快、爽点密集、人设鲜明。允许中二热血的台词和轻松玩梗，对话活泼幽默，每章至少一个'打脸/震惊全场'的爽点。"

## 5. Prompt 设计原则

### 5.1 策划
> "你是一位资深出版策划，擅长为长篇小说设计世界观、人物弧光与分章节奏..."

针对 **热血磅礴风** 策划需特别强调：
> "世界观要横跨宏大时空，设置多层上古秘辛，主角身世需埋设深远伏笔，配角名号要响亮霸气。"

针对 **10后向** 策划需强调：
> "人设要标签鲜明（如系统/签到/满级），前三章必须出现一个高能爽点，节奏要快，避免冗长背景铺陈。"

### 5.2 作者
> "你是一位职业网络小说作家，擅长 {expandedStyle}。请根据以下大纲与上下文，撰写第 N 章初稿（约 3000 字）。要求：情节紧凑、人物动机清晰、对话自然、埋好伏笔。"

### 5.3 编辑
> "你是一位资深文学编辑，专精长篇小说结构审查。请阅读以下第 N 章初稿，并从以下维度给出修改意见：
> 1. 情节是否与上一章连贯？
> 2. 人物行为是否符合已有设定？
> 3. 节奏是否拖沓或跳跃？
> 4. 是否有逻辑漏洞或伏笔未收？
> 请输出：【总体评价】+ 【具体问题清单】+ 【修改建议】。"

### 5.4 去AI化
> "你是一位风格化写手，任务是去除文本中的'AI腔'。请做到：
> 1. 句式长短错落，避免排比套话；
> 2. 对话口语化，符合人物身份；
> 3. 修辞更具体（少用'非常'、'很'、'仿佛'等模糊词）；
> 4. 保留并强化 {expandedStyle} 的风格特征。
> 请直接输出去AI化后的完整章节。"

### 5.5 校编
> "你是一位文字校对编辑，请修正以下文本中的错别字、标点误用、语法错误和格式问题。不要改变原文的叙事风格和情节内容。"

### 5.6 读者
> "你是一位忠实读者，刚读完以下小说章节。请从读者体验角度给出反馈：
> 1. 可读性评分（1-10）
> 2. 本章爽点（最吸引你的地方）
> 3. 本章毒点（让你觉得别扭或想弃书的地方）
> 4. 期待感（对接下来的情节期待程度 1-10，并说明原因）
> 输出为结构化 JSON。"

**10后读者** 的反馈维度可额外增加：
> 5. "中二燃度"（1-10）
> 6. "梗密度是否合适"
> 7. "是否希望立刻看到下一章"

## 6. 模型配置建议

每个角色可独立配置模型，默认推荐：

| 角色 | 默认模型 | 理由 |
|------|----------|------|
| 策划 | `deepseek-r1` | 需要强逻辑推理与规划能力 |
| 作者 | `deepseek-v3` | 创作力与长文本生成稳定 |
| 编辑 | `deepseek-r1` | 审查逻辑与一致性 |
| 去AI化 | `hunyuan` | 文笔优美，风格化能力强 |
| 校编 | `deepseek-v3` | 细致、出错率低 |
| 读者 | `deepseek-v3` | 理解力强，反馈结构化 |
| 摘要 | `deepseek-v3` | 提取精准 |

## 8. 性能与成本控制

每章 `multi-agent` 模式需要 **6-7 次** LLM 调用（作者、编辑、作者改、去AI化、校编、读者、摘要）。

优化建议：
1. **可跳过步骤**：在 `meta.json` 中设置 `skipSteps: ['reader', 'edit']` 以快速迭代。
2. **编辑内联**：编辑不输出独立文件，而是直接作为 prompt 让作者改稿时内化，减少一次调用。
3. **异步批处理**：Web UI 中可勾选"只生成到 final，跳过 feedback"来节省流量。

## 9. 后续扩展

- **Agent 辩论机制**：引入 2 位不同风格的编辑，分别提出意见，作者综合修改。
- **角色专属记忆**：为每个主要角色维护独立记忆文件（如 `memory_writer.txt`、`memory_humanizer.txt`），记录该角色在不同章节中的风格偏好与修正历史。
- **读者反馈驱动**：将 `reader_feedback_N.json` 中的毒点自动汇总为下章 prompt 的"避坑清单"。
- **风格微调**：针对热血磅礴风与 10后向 收集更多样例，训练专属 few-shot prompt 模板。
