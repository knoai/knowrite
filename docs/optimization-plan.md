# 长篇小说生成优化方案

## 1. 章节字数控制

### 目标
- **每章约 2000 字**，允许浮动范围 **1800–2200 字**。
- 硬性下限 1600 字，上限 2500 字，避免过短（信息密度不足）或过长（节奏拖沓）。

### 实现方式
- 在 **Writer Prompt** 与 **Pipeline Prompt** 中明确注入字数指令：
  > "撰写第 N 章初稿（约2000字，允许范围1800-2200字，禁止低于1600字或高于2500字）"
- 若模型持续超字数，可在 Editor Prompt 中增加结构审查项：
  > "□ 字数：本章是否控制在 2000 字左右？是否存在可删减的冗余描写？"

---

## 2. 卷结构设计

### 目标
- 每卷 **4–6 万字**，对应 **20–30 章**。
- 每章约 2000 字，全书按卷数自动推算总篇幅。

### 实现方式
- `generateMultiVolumeOutline` Prompt 中强制声明：
  > "全书包体量为 ${totalVolumes * 5} 万字左右，每卷 4-6 万字，含 20-30 章，每章约 2000 字。"
- `generateVolumeOutline` Prompt 中同样约束：
  > "本卷规模为 **20-30 章，4-6 万字**，每章约 2000 字。"
- 在 `meta` 中记录每卷的 `plannedChapters`（规划中）与 `chapterRange`（实际已写），便于前端展示进度。

---

## 3. 滚动上下文压缩与智能保留

### 问题
- 长篇小说写到第 20+ 章时，若仍将"前 1 章全文 + 前 3 章摘要"全部注入 Prompt，上下文过长，费用高且稀释焦点。
- 摘要窗口过小（仅 3 章）会导致中期设定、支线伏笔被遗忘。

### 优化方案

#### 3.1 前一章全文压缩
| 策略 | 说明 |
|------|------|
| 第 2–4 章 | 保留前一章**完整正文**（情节尚短，全文不会超标） |
| 第 5 章起 | 前一章由 AI 压缩为 **300–500 字的情节提要**，缓存为 `chapter_N_compressed.txt` |

**压缩维度**：
1. 核心情节（发生了什么，主角做了什么关键选择）
2. 情绪转折（本章高点/低点）
3. 悬念状态（结尾留下的未解问题）
4. 关键对白（仅保留 1–2 句对未来有直接推动的台词）

#### 3.2 摘要智能分层
- **近史摘要**：保留最近 **5 章**的完整摘要（SUMMARY_WINDOW_SIZE = 5）。
- **远史摘要**：5 章之前的所有摘要，由 AI 合并压缩为一段 **400–600 字的"远史提要"**，缓存为 `context_distant_summary_1_X.txt`。
  - 只保留对主线仍有影响的情节、设定、未解悬念。
  - 已解决且无后续影响的小支线忽略。
  - 突出人物关系变化和世界观关键规则。

#### 3.3 效果
- 写到第 30 章时，上下文长度大致为：
  - 1 段远史提要（~500 字）
  - 5 段近史摘要（每段 ~150 字，共 ~750 字）
  - 1 段前一章压缩提要（~400 字）
  - 总计约 **1600 字**，可控且信息密度高。

## 4. 分卷低耦合与切地图

### 目标
- 每一卷尽量**与上一卷关联少**，降低读者追更/续写的认知负担。
- 每一卷尽量**切地图**（进入新舞台、新势力范围、新规则环境）。

### 实现方式

#### 4.1 多卷架构 Prompt 约束
```
分卷设计原则（必须严格遵守）：
- 尽量切地图：每一卷应进入相对独立的新地图/新舞台/新势力范围。
- 卷间低耦合：每一卷的核心冲突在本卷内基本闭环解决，与上一卷的遗留问题关联尽量少。
- 卷末钩子：最好是“新地图入口/新身份/新敌人”型钩子，而非“前卷悬念未解”型钩子。
```

#### 4.2 逐卷纲章 Prompt 约束
```
- 切地图原则：本卷的主要舞台/地图应与前卷有明显区隔。
- 卷内闭环：本卷核心冲突在本卷内基本解决，不大量遗留未结支线给下一卷。
- 卷末必须留下强钩子（新地图入口/新身份/新敌人）。
```

#### 4.3 代码侧支持
- `continueNovel` 支持 `targetVolume` 参数，允许用户直接切换到新卷续写。
- Web UI 与 CLI 均提供**卷选择器**，减少跨卷误操作。
- `getCurrentVolumeOutline()` 保证写章节时只读取**当前卷**的纲章，避免前卷内容混入。

## 5. 防重复机制

### 5.1 智能记忆索引 (`memory_index.json`)
- 每章写完后，AI 从摘要中提取 **entities（实体）/ plot_threads（情节线）/ rules（规则）**。
- 建立词条 → 出现章节列表的索引。

### 5.2 防重复提醒 (`buildAntiRepetitionReminder`)
- 写新章前，扫描当前卷纲章关键词。
- 若某词条在**当前滚动窗口之外**已出现过 **≥2 次**，则生成提醒：
  > "以下内容已在前文多次详细交代，本章禁止再次长篇解释，只需一句话带过：
  > - XX境界（首次出现于第3章，最近在第12章已有交代）"

### 5.3 与上下文压缩的协同
- 远史提要只保留"仍有影响"的信息，天然过滤掉大量已解决的冗余细节。
- 近史摘要保证短期情节的严谨衔接。
- 前一章压缩提要确保人物状态、悬念、对白的精确传递。

## 6. 待进一步验证项

| 项 | 验证方式 |
|---|---|
| 2000 字约束是否被模型稳定遵循 | 观察后续生成章节长度分布 |
| 压缩质量是否影响情节连贯性 | 对比保留全文 vs 压缩提要的续写一致性 |
| 切地图/低耦合是否被大纲 Agent 执行 | 人工抽查 `outline_multivolume.txt` 和 `volume_X_outline.txt` |
| 远史提要缓存是否命中 | 检查 `context_distant_summary_*.txt` 生成频率 |
